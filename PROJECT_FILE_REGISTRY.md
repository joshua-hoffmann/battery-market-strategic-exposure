## V2.2 File Governance Rule

If V2.2 proceeds, use V2.2-specific filenames.

Preferred naming pattern:
- data/processed/critical_minerals_processing_refining_snapshot.csv
- outputs/briefs/processing_refining_concentration_snapshot_brief.md
- outputs/charts/processing_refining_concentration_snapshot.png
- src/calculate_processing_refining_snapshot.py

Do not overwrite V2.1 files unless intentionally regenerating V2.1.

## Duplication Rule

Do not create a new file if an existing file already serves the same purpose.

Examples:
- Do not create another V2.1 concentration metrics file if critical_minerals_concentration_metrics.csv already serves that purpose.
- Do not create generic output names such as final.csv, results.csv, report.md, or chart.png.
- Do not create scripts/ because src/ already exists.
- Do not create results/ because outputs/ already exists.
